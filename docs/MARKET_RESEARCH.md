## Потребности пользователей (из форума)

### Проблемы

1. **Удалённые объекты** — 12+ км в сопку, 100+ км от центра
2. **Сложный доступ зимой** — невозможно оперативно диагностировать
3. **Нет единого мониторинга** — куча отдельных окон/компов
4. **Интеграция** — нужна связь с Zabbix и другими системами
5. **Сертификация** — вопросы по самописному ПО

### Запрошенный функционал

- Мониторинг состояния всех РС в сети
- Удалённая смена частоты
- Управление ПШ (подавитель шума)
- Диагностика неисправностей
- Звуковая сигнализация аварий
- Исторические архивы событий
- Мнемосхемы

## Потенциальные заказчики

1. **Госкорпорация по ОрВД** — централизованное решение для всех центров
2. **Региональные центры ОВД** — для своих объектов
3. **Аэропорты** — для собственного оборудования

## Конкурентные преимущества RCMS-GA

1. **Открытый исходный код** (GPL-3.0) — доверие, аудит безопасности
2. **Linux-нативное** — совместимость с Astra, РЕД ОС
3. **Модульная архитектура** — легко добавлять новое оборудование
4. **Профессиональный GUI** — Qt, современный интерфейс
5. **Документация на русском** — для российских специалистов

## Монетизация (варианты)

1. **Поддержка и внедрение** — платные услуги при бесплатном ПО
2. **Сертификация** — получение сертификата, продажа сертифицированной версии
3. **Кастомизация** — доработка под конкретного заказчика
4. **SaaS** — облачный мониторинг (если допустимо по безопасности)
